네트워크
    - 통신 : 정보를 서로 주고받는 과정 및 행위
        - 모스부호, 신호등, 편지, 신문, 채팅, 이메일 등

    - 네트워크 : 인터넷을 통해 다양한 정보외 서비스를 주고받는 과정 및 행위
    - 특징
        1. 글로벌 연결성 : 전 세계의 컴퓨터들과 연결하므로 지리적 제약이 없고 정보 공유가 가능
        2. 분산 구조 : 중앙집중식이 아니므로 일부분 장애가 발생해도 전체 시스템에 영향을 주지 않는다
        3. 다양한 서비스 : 이메일, 웹서비스, 스트리밍, 파일전송 등
        4. *프로토콜 : 표준화된 통신 규약으로 서로 다른 시스템간의 원활한 통신 제공
    - 구성 요소
        1. 네트워크 인프라 : 케이블, 라우터, 위성 등등 물리적 장치와 연결매체 필요
        2. *프로토콜 : 인터넷 데이터 통신 규칙 및 절차
        3. 서버 : 데이터 제공 컴퓨터 (톰캣, MySQL DB 등)
        4. 클라이언트 : 데이터 요청 및 수신 컴퓨터 (브라우저 등)
        5. 도메인(DNS) : 복잡한 IP구조를 문자열로 변환한 주소 제공 (https://www.naver.com/ 등)

프로토콜이란?
    - 네트워크에서 데이터를 교환하기 위한 규칙과 절차
    - 주요 프로토콜
        1. IP : 네트워크 컴퓨터간 통신 시 서로를 식별
            - 집 주소
            - e.g. 192.168.30.1 (공개용 IP)
            - 0~255 조합 4자리, IPv4 vs IPv6
            - 내 IP 확인 : cmd > ipconfig
            - 로컬IP(현재PC IP) : localhost == 127.0.0.1
        2. PORT : 네트워크 컴퓨터 내부 프로세스(프로그램) 식별
            - 집 안의 특정 방 문 식별
            - 서버PC의 프로세스들 중 어떤 프로그램과 연결할지 식별
            - 톰캣 디폴트 : 8080, MySQL : 3306, HTTP : 80 등
        3. *HTTP : 웹 브라우저와 웹 서버간 데이터 전송
        4. SMTP : 이메일 송수신
        5. FTP : 파일 전송
        6. DNS : 도메인 네임 시스템, IP주소를 문자형식으로 바꿔주는 시스템
        7. HTTPS : SSL/TLS가 적용된 보안 HTTP
        8. SSL/TLS : 인터넷을 암호화해서 보안 제공하는 역할
    - 구조
        1. OSI 7계층 모델
            송신, 수신 호스트) 응용 <> 표현 <> 세션 <> 전송 <> 네트워크 <> 데이터 링크 <> 물리

            응용 계층 : 사용자와 직접 상호작용 (HTTP, FTP, NFS, SMTP, Telnet)
             표현 계층 : 데이터 형식 규정 (JPEG, SMB, AFP, CDR, MPEG)
              세션 계층 : 인증 및 서비스 (TLS, SSH, ISO 8327, RPC)
               전송 계층 : 프로세스간 데이터 전송 (TCP, UDP, RTP, SCTP, SPX)
                네트워크 계층 : 데이터 경로설정 (라우팅, 스위칭) (IP, ICMP, IPX, ARP, BGP)
                 링크 계층 : 네트워크 기기간 데이터 전송 (Ethernet, ATM, LAN, WIFI)
                  물리 계층 : 시스템간 물리적 연결, 전기 신호 변환 (Modem, Cable, Fiber)

        2. TCP/IP 4계층 모델
            응용 > 전송 > 네트워크 > 링크 > 물리 >  > 물리 > 링크 > 네트워크 > 전송 > 응용

            응용 계층 : 사용자 APP 서비스 제공 (HTTP, FTP 등)                        [응용-표현-세션계층]
            전송 계층 : 데이터 전송의 신뢰성 보장 (TCP/UDP)                           [전송계층]
            인터넷 계층 : 데이터 패킷의 주소 & 경로 지정 (IP)                         [네트워크 계층]
            네트워크 액세스 계층 : 물리적 네트워크 연결 (OSI의 물리 + 데이터 링크 계층)  [링크-물리계층]
