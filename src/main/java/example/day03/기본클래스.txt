    2. Object
        - 자바의 최상위 클래스 (모든 클래스들의 조상 : Object를 상속받는다)
        - 모든 클래스들 extends Object
            -> JAVA : 100% 객체지향을 구현하기 위해 설계됨
        - 주로 쓰이는 메소드
            1. toString() : 객체를 문자열로 표현해서 반환, 오버라이딩으로 설명 등 반환

            2. equals(Object 객체) : 두 객체가 동일한지 여부를 반환, 오버라이딩해서 동일함을 정의 가능
                > == vs .equals()
                    - == : 스택 메모리(주로 지역변수)가 가지는 값(리터럴)/참조값 비교, 주로 기본타입 비교시 사용
                    - equals : 힙 메모리가 가지는 참조값을 비교, 주로 참조타입 비교시 사용
                        - 10 == 10 : true
                        - 'A' == 'A' : true
                        - new String("A").equals(new String("A")) : false, new 사용시 서로 다른 참조값(주소) 생성되므로 두 "A"는 다르다

            3. hashCode() : 객체의 해시코드(주소값) 반환
                - 해시(hash) : 정보 저장 또는 검색시 사용하는 자료구조(자료저장방법) 중 하나
                - 정보를 어디에 저장할 것인지, 어디서 가져올 것인지 해시 함수를 사용하는 방법
                - 해시 함수 : 객체의 특정 정보를 매개변수 값으로 넣으면 저장될 위치나 저장된 해시테이블주소(위치) 반환

            4. clone() : 객체를 복제해서 동일한 멤버변수를 가진 새로운 객체를 반환
                - Cloneable 인터페이스를 implements 구현해서 구현클래스 이용한 clone() 메소드 재정의/오버라이딩해서 사용가능
